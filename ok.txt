=== DOCKERFILE ===
FROM nginx:alpine
# Copia todos os arquivos do projeto para o diretório do servidor
COPY . /usr/share/nginx/html
# Substitui a configuração padrão do Nginx pela nossa otimizada
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

=== NGINX.CONF ===
server {
    listen 80;
    server_name localhost;
    include /etc/nginx/mime.types;

    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # Configuração CRÍTICA: Faz o navegador tratar o .tsx como JavaScript
    location ~* \.tsx$ {
        root /usr/share/nginx/html;
        add_header Content-Type application/javascript;
    }

    # Cache para arquivos estáticos
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|json)$ {
        root /usr/share/nginx/html;
        expires 1y;
        add_header Cache-Control "public, no-transform";
    }
}

=== COMO RODAR ===
1. Certifique-se de que os arquivos index.html, index.tsx, Dockerfile e nginx.conf estejam na mesma pasta.
2. No Easypanel, ao apontar para o repositório, ele usará o Dockerfile automaticamente.
3. O Babel no index.html transformará o código do index.tsx em tempo de execução.
